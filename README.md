# Blog-Skills

Skills to make blog easy and fast.

## Overview

Blog-Skills is a small collection of markdown "skill" documents that provide templates, formatters, and content-generation utilities to speed up writing and publishing blog posts. The repository is organized as a set of focused skill files you can read, adapt, and integrate into your blog authoring workflow.

## Contents

- [Component formatter skill](https://github.com/Vishwas1345/Blog-Skills/blob/main/Component-formatter-skill.md) — Tools and templates to format UI components or code snippets consistently for blog posts.
- [Content skill](https://github.com/Vishwas1345/Blog-Skills/blob/main/Content-skill.md) — Guidance and templates for generating and structuring blog content (outlines, sections, SEO metadata).
- [HTML skill](https://github.com/Vishwas1345/Blog-Skills/blob/main/HTML-skill.md) — HTML/templating helpers and examples for converting content into web-ready markup.

> Repository: [Vishwas1345/Blog-Skills](https://github.com/Vishwas1345/Blog-Skills)

## Who is this for

- Bloggers who want repeatable templates and helpers for writing and publishing.
- Developers building content automation, static-site pipelines, or post-processing scripts for blog material.
- People who want example patterns for consistent formatting and HTML output.

## How to use

1. Browse the skill files listed above and pick the one that fits your need.
2. Copy the templates/snippets into your authoring tool, static-site generator, or editor macros.
3. Adapt the examples to your tooling (e.g., convert markdown snippets into your site's partials or components).

Typical workflows:
- Use the Content skill to produce an outline and draft.
- Use the Component formatter to ensure code blocks and UI examples are consistently presented.
- Use HTML skill to convert the draft into site-specific templates.

## Contributing

Contributions are welcome.

- Open an issue to propose new skills, request improvements, or report problems.
- Send a pull request that:

  - Follows the style of existing files: clear headings, examples, and usage notes.
  - Includes examples and, when relevant, small copy-pasteable snippets.

Suggested PR checklist:
- [ ] Add or update a skill markdown file in the repository root.
- [ ] Keep examples concise and runnable where applicable.
- [ ] Update this README if you add a top-level skill that changes the repository scope.
